1×0 对设备进行无辅助解锁

为什么叫做无辅助解锁，因为除了你要降级、Root，没有其他外界干涉因素，包括淘宝奸商:)

 

1×1 预先工作 - 系统降级

使用「HiSuite」对设备进行「降级」操作，具体行动:

  1. 打开「HiSuite」，建立连接之后，打开「系统升级」
  2. 待获得数据完成后，进入「其他版本」页。
  3. 回退到上一个版本。
  4. 反复进行如上操作，直到无法再回退或者该版本可以临时Root
  5. 结束。

 

1×2 临时Root

声明：Root行为本身存在风险，且可能会污染系统。使用何种工具、攻击脚本，本人不做推荐；本人也不可能对任何的数据丢失、系统BootLoop负担任何责任。

这一步不再说什么，只要降级正确，一定有一个版本可以获得临时Root。

使用任意工具、攻击脚本获得临时Root，然后继续。

 

1×3 备份NVME位设备

谈到nvme这个东西，我不得不说两句。

在Linux下，这不是一个系统文件，而是一个可以Dump成数据的设备

请不要再说这是个文件了，各位××破解、×粉论坛的老哥，窝球球乃们了QuQ

在ADB Shell执行以下命令：

$ su
# mkdir /sdcard/nvme-dmp
# cd /sdcard/nvme-dmp
# dd if=/dev/block/platform/ff1fe000.dwmmc0/by-name/nvme of=nvme-dmp.img

如果看到有且只有 类似如下内容：

xxxxxxxxxx
记录了×××+0 的读入
记录了×××+0 的写出
×××× bytes copied, ×.×××××× s, ×.× kB/s

证明nvme设备备份成功。

1×5 Fastboot解锁

在有fastboot/fastboot.exe的命令行下输入：

xxxxxxxxxx
$ adb oem unlock 1145141919198101
# 这串恶臭的数字代表你获得的16位解锁码，不是李田所先生的电话号码>_<

2×3 OpenKirin 刷写

对于OpenKirin这个特立独行的家伙，你用自己准备的CW Recovery或者Team Win Recovery Project绝对刷进去不能开机。

所以我们需要用的是Fastboot + 华为原厂Recovery的迫真黄金搭档啦。

手机重启进Fastboot 模式。确认显示的是红色UNLOCKED那么请你继续看。

如果你在期间进行了升级，无论如何，请回到章节1×5， 尝试再次解锁，不然你肯定会碰壁。

因为在每次OTA升级之后，升级脚本一定会对System这样重要的分区进行回锁。

 

在有Fastboot/Fastboot.exe的命令行内运行如下命令：

​x
$ fastboot flash system ~/openkirin/rr-beta1.img
# 此处，~/openkirin/rr-beta1.img 需要替换成你的ROM解压目录
# Trick：无论什么操作系统，直接拖动资源管理器中的ROM image到命令行中，会自动输入绝对路径。
​
# 等待刷写全部阶段完成，重启手机。
$ fastboot reboot

 

别走，没完事！还差一步！

在启动时，一旦出现「设备已经被解锁且不再被信任」字样，按住 音量+按键 三秒，进入Huawei eRecovery，此时会提示你「Data分区似乎损坏，需要抹掉数据来修复」，点击「抹掉」，完成后点击「恢复出厂设置」，并点「是」进入Huawei Recovery Ramdisk。

出现红色感叹号也别慌，要做的已经完成了，但是别着急开机进入配置向导，先关机，看2×4！

 

2×4 OpenKirin 缺陷与解决方案

没错，一群大佬整出来的东西也有问题。

比如Data分区强制加密这一机制，目前真的不能只依赖华为的官方Recovery Ramdisk解决。

所以，需要使用TWRP/TWRP 7to(奇兔) Fork来解决强制加密这个让机器无法开机的玄学问题。

进入Fastboot，在有Fastboot/Fastboot.exe的命令行内运行如下命令：

xxxxxxxxxx
$ fastboot flash recovery_ramdisk ~/openkirin/twrp.img
# ！！！！！千万不要写成flash recovery，你打死也写不进去！！！！！
#  ！！！！！！！！注意是flash recovery_ramdisk！！！！！！！！
# 此处，~/openkirin/twrp.img 需要替换成你的Recovery解压目录
# Trick：无论什么操作系统，直接拖动资源管理器中的Recovery image到命令行中，会自动输入绝对路径。
​
# 等待刷写全部阶段完成，重启手机。
$ fastboot reboot

在手机断电准备再次开机时，按住音量+，进入刷写的Team Win Recovery Project/奇兔TWRP Fork。

进入「Wipe / 抹掉」，点击「Wipe Data / 清除Data分区」，输入yes，按下回车，进入抹掉状态。

如果是奇兔TWRP Fork，可以在「其他」里面找到「解除Data分区强制加密」，滑动执行后，直接启动到新ROM。

 

2×5 OpenKirin 其他注意事项

  • 当你需要升级的时候，相当于重新走了一次2×3-2×4，刺激不? >_<
  • 当完成2×4时，写入Magisk只需要按照正常程序刷入install.zip就可以正常工作，不需要看官网的刷法。
  • 在测试的Huawei Mate 8 公开版上，只有卡槽2以2.5G/EDGE频率工作，而不是正常的双卡全部工作

。其他存在这种奇妙现象的设备，可以给我写信告知。

  • 在测试的ROM：Emotion UI 8 Based ROM - Onmi Pie Beta 5.2中，存在WLAN锁屏时无告知断开，自我重启的毛病，即使修改了Captive Server为大陆境内服务器也没有改善。如果不能接受就别刷Pie(9.0)了，换更低的Oreo(8.0)。
  • 使用AOSP系列ROM后，发热和耗电可能没有EMUI控制的好，建议借助绿色守护 + Riru + EdXposed解决后台问题。Pie(9.0)这种问题很少因为是暴政w
